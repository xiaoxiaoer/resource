# 停车券业务审核 Skill

## 描述

对停车券业务进行逐项审核。将上传的车场资料数据与 BEM 系统数据进行对比分析，按检查项输出初审结果。

## 触发条件

- 资料完整性检查通过 + 业务类型为"停车券业务"时触发

## 前置依赖

- `completeness-check` 已通过
- `bem-fetch` 已获取 BEM 数据（临停收入、月票收入、月票类型）
- `company-lookup` 已获取企查查数据（参保人数）

## 输入

| 数据 | 来源 |
|------|------|
| 上传的车场资料 | 运营上传，由 Harness 解析 |
| BEM 临停收入数据 | bem-fetch skill |
| BEM 月票收入数据 | bem-fetch skill |
| BEM 月票类型数据 | bem-fetch skill |
| 企查查参保人数 | company-lookup skill |

## 检查项清单

### 1. 临停收入对比

| 对比项 | 资料来源 | BEM来源 | 判定 |
|--------|---------|---------|------|
| 月均临停收入 | 测算表-车场月均临停收入 | 临停月均(实收金额-微信) | 一致✅ / 不一致⚠️输出差异数据 |

若 BEM 数据有下滑趋势，额外输出趋势说明。

### 2. 月票收入对比

| 对比项 | 资料来源 | BEM来源 | 判定 |
|--------|---------|---------|------|
| 月均月票收入 | 测算表-车场月租收入 | 月票月均收入 | 一致✅ / 不一致⚠️输出差异数据 |

若无涉及置换月票则无需对比。

### 3. 合同有效年限检查

- **规则**：合同有效年限（月）必须在承包期限内
- **判定**：范围内✅ / 超出❌

### 4. 券消耗年限检查

- **规则**：券消耗年限（月）必须晚于合同有效年限（月）
- **判定**：满足✅ / 不满足❌

### 5. 自有支付渠道 — 可消耗收入占比计算

**触发条件**：车场存在自有小程序/APP/ETC

```
可消耗收入占比 = [总计 - 微信小程序（开放平台） - ETC] / 总计
```

数据来源：bem-fetch 的临停收入各渠道明细。

### 6. 停车券月均消耗额度检查

**触发条件**：存在自有支付渠道

- **规则**：停车券月均消耗额度 ≤ 可消耗收入占比 × 车场月均临停收入
- **判定**：满足✅ / 不满足⚠️输出差额

### 7. 分期代扣 / CRM专项服务费 / 业务费

- **规则**：有则标出，不自动判定
- **输出**：📋 标注"需运营人工评估" + 具体金额

注意事项：
- CRM专项服务费与业务费不会同时共存
- 若有设备、服务置换金额的填写，则保留 CRM 专项服务费的值

### 8. 毛利率计算

```
毛利率 = 项目整体毛利 / 停车券总价值
```

- 输出毛利率百分比
- 结合实际采买折扣输出对应权限范围

### 9. 合作客户主体 — 企查查

- 输出参保人数
- 承包方时额外检查：承包到期时间是否超出合同时间

## 输出格式

按 `prompts/output_template.md` 统一格式输出，逐项给出 ✅/⚠️/❌/📋 标记。
