# 资料完整性检查 Skill

## 描述

检查运营上传的车场资料是否所有必填字段均已填写。
纯 AI 逻辑，不依赖外部脚本。

## 触发条件

- 运营上传车场资料后自动触发
- 运营补充资料后再次触发

## 输入

从上传的文件中解析出的结构化数据，字段定义参考 `templates/fields_required.json`。

## 执行步骤

### 1. 识别业务类型

根据上传资料内容判断：
- 停车券业务：资料中包含停车券相关信息
- 车位置换业务：资料中包含车位置换相关信息
- 无法判断：询问运营

### 2. 字段完整性检查

根据业务类型，加载对应必填字段列表，逐项检查：

**通用必填字段（两种业务共有）：**
- 合作客户主体
- 是否产权方/承包方
- 承包到期时间（承包方时必填）
- 合同有效年限（月）
- 是否存在自有小程序/ETC支付渠道
- 车场名称
- 车场月均临停收入
- 车场月均月票收入（涉及月票置换时必填）

**停车券业务额外字段：**
- 券消耗年限（月）
- 停车券月均消耗额度
- 停车券总价值
- 项目整体毛利

**车位置换业务额外字段：**
- 设备服务置换金额
- CRM结算价
- 对外办理的月卡费用
- 是否允许张贴物料

### 3. 输出检查结果

```
## 资料完整性检查结果

业务类型：[停车券业务 / 车位置换业务]

✅ 已填写字段（N项）：
  - 合作客户主体：XXX公司
  - ...

❌ 缺失字段（M项）：
  - 合同有效年限（月）
  - ...

请补充以上缺失字段后重新上传。
```

## 注意事项

- 字段值为空、"无"、"未填写" 等均视为未填写
- 承包到期时间仅在"承包方"时必填，产权方时可选
- 自有小程序/ETC 勾选"是"时，需额外检查相关渠道收入数据
